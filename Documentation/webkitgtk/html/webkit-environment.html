Logging

You can make WebKitGTK+ log some of what it is doing internally by using the WEBKIT_DEBUG environment variable. It takes a space-separated list of names, representing the various aspects of the work the library performs. Notice that most of the logging is only done in debug builds (that you get when you pass --enable-debug to the configure script).

A list of possible values for the environment variable follows:

BackForward

Editing

Events

Frames

FTP

History

IconDatabase

Loading

Media

Network

NotYetImplemented

PageCache

PlatformLeaks

Plugin

PopupBlocking

SpellingAndGrammar

SQLDatabase

StorageAPI
